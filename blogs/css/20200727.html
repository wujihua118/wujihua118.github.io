    # 什么是伪类和伪元素、以及二者的区别

    • 伪类：一种css选择器的关键字，符合当前伪类对应状态将会应用其对应的方式
      • 例如：#app:hover { color: red } 当鼠标悬停在id为app的标签上，该标签将会变成红色
    • 伪元素：附加到选择器末尾的关键词，允许对伪元素选中的特定部分修改其样式
      • 例如：p::first-line { color: red } 命中的是所有p标签的第一行元素，将会变为红色，如果p标签内有多行，则除了第一行其他行都不会被命中
    • 区别：
      • 伪类单冒号，伪元素双冒号
      • 伪类命中的是完整dom 元素，伪元素命中的是某个dom元素的一部分
      • 伪类不能插入元素，伪元素可以，但插入元素不在dom树中

    # 两种盒模型

    • 标准盒模型：box-sizing: content-box width=内容宽度（默认使用）
    • IE盒模型：box-sizing: border-box width=内容+padding+border

    # 相对定位与绝对定位

    • 相对定位 position: relative 不脱离文档流，参考自身初始位置使用top，left等进行定位，初始位置保持占位，可以使用z-index进行分层
    • 绝对定位：position: absolute 脱离文档流，参考最近一层设有定位属性的父级进行定位，如果无父级，则参考body远点定位，可以使用z-index分层

    # css选择器

    • 标签选择器p {}
    • 类选择器.p {}
    • ID选择器#p {}
    • 伪类选择器:hover {}
    • 全选择器* {}
    • 后代选择器div p {} 选中div标签下所有p标签元素
    • 子选择器div>p {} 选中div标签下第一层子元素所有p元素
    • 相邻选择器div+p {} 选中与div同级往后第一个p元素，不是p元素不选中
    • 属性选择器a[title] {} 选中设置了title属性的a标签

    # 实现自适应浏览器宽高正方形

    • 使用vh/vw：100vw = 当前窗口宽度（html根元素宽度），相当于将窗口宽度分为100份，所以只需要将正方形宽高设置为相同的vw即可

      .square { width: 50vw; height: 50vw; }

    • 使用padding：当前元素padding的百分比是按照其父元素宽度计算，所以只需要设置当前元素宽度百分比与padding-top/padding-bottom相同即可（当前元素height最好设置成0，防止内容填充引起高度变化）

      .square {width: 50%; padding-top: 50%; height: 0; }

    • 使用伪元素：使用after在当前元素后添加伪元素，伪元素设置为块级元素，设置其padding-top高度为100%（伪元素padding百分比按照当前元素宽度计算），使用伪元素撑开当前元素高度

      .square {
  width: 50%;
}
.square::after {
  content: '';
  display: block;
  padding-top: 100%;
}

    # ul除了最后一个li其他设置有边框效果

      ul li {
  border-right: 1px solid red;
}
ul li:lastchild {
  border-right: none;
}

    • 使用伪元素嵌套ul:not(:last-child)

      ul:not(:last-child) {
  border-right: 1px solid red;
}

    # link与@import区别

    • link <link rel="stylesheet" type="text/css" href="style.css"/>
    • @import <style>@import(style.css)</style>
    • 区别：
      • link标签除了可以导入css文件还可以其他文件，@import只可以导入css文件
      • link标签在页面加载时同时加载css，而@import则在页面加载完成之后加载css
      • link标签在head标签内，而@import在style标签内或者在css样式中

    # 常用的css单位

    • px：像素，不同分辨率px个数不一样，比如分辨率100*100则横着100像素竖着100像素
    • em：相对于使用em的dom元素它的父元素大小计算。比如<div style="font-size: 10px"><p style="font-size: 2em"></p></div>，父元素div的字体大小为10px，那么1r=em就为10px，所以p元素字体大小2em实际为20px
    • rem：相对于使用em的dom元素它的根元素（html）字体大小计算。比如<html style="font-size: 10px"><p style="font-size: 2em"></p></html>，html根元素字体大小为10px，那么1r=em就为10px，所以p元素字体大小2em实际为20px
    • vh：1vh为视口高度（审查元素里面的html高度）1%
    • vw：1vh为视口宽度（审查元素里面的html宽度）1%

    # 实现动画的几种方式

    • 使用伪类:hover，该方式无过渡

      .app {
  background-color: red;
  width: 100px;
}
.app:hover {
  width: 200px;
  background-color: blue;
}

    • 使用transition：该方式一般配合:hover、:active等伪类一起使用。它由四个合成属性组成，可以控制需要动画的属性，控制动画过渡的时间，控制动画过渡的方式（匀速、先快后慢等），控制动画延迟时间。

      • transition-property：控制需要动画的属性

        • transition-property:all：所有可以进行动画的属性都进行动画
        • transition-property:none：无动画效果
        • transition-property:width,height...：自定义需要 进行动画的属性

      • transition-duration：控制动画过渡的时间

        • transition-duration:1s：动画在1s内过渡完成
        • transition-duration:1s,2s,3s...：按照顺序对应transition-property三个属性分别的动画过渡时间

      • transition-timing-function：控制动画的过渡方式

        • transition-timing-function:linear：动画匀过渡
        • transition-timing-function:ease：默认值，慢速开始，中间变快，慢速结束
        • ...

      • transition-delay：控制动画延迟开始的时间

        • transition-delay:3s：等待三秒，动画开始
        .app {
    background-color: red;
    width: 100px;
    transition: all 1s linear 3s;
}

.app:hover {
    width: 200px;
    background-color: blue;
    transition: all 1s linear 3s;
}

    # 移动端1px问题及解决方案

    • 1px问题

      pc端的dpr为1，不会出现1px的问题。dpr = 物理像素/css像素

      物理像素：实际屏幕发光的小点，css像素：常设置的1px

      dpr为1就意味着1个物理像素对应着1个css像素

      但是移动端的dpr可能为1，2，3等等。屏幕越高清，dpr越高。当dpr为2时，设置1px在移动端将对应着2个物理像素（或者2个屏幕发光点）。所以移动端设置1px就会变粗（因为有2个物理像素在发光）。

    • 解决方案

      • 直接使用0.5px。假设当前移动端dpr为2，意味着设置1px将对用2个物理像素。要实现1个物理像素，可以设置0.5px。该方法兼容性不好：Google/Firefox会将0.5px转换成1px。

      • 使用伪元素+绝对定位+缩放。比如原先我们设置border-top:1px solid black，现在我们要做的是取消border设置，加入伪元素高度1px，然后进行绝对定位放置在border-top位置，同时使用scaleY(0.5)将伪元素垂直方向缩放原来的0.5，就可以使得1px变成0.5px。

      • 设置meta标签缩放页面。比如现在我们页面dpr为2，我们设置1px将对应2物理像素，那我们将我们的页面进行缩放原先的1/2，此时我们设置的1px实际上被缩放至0.5px，0.5px对应1物理像素，所以此时我们写的1px === 1物理像素。

        const dpr = window.devicePixelRatio;
const meta = document.createElement('meta');
meta.setAttribute('name','viewport');
// 宽为设备宽度，禁用用户缩放，默认缩放1/dpr，最大缩放比例1/dpr，最小缩放比例1/dpr
meta.setAttribute('content',`width=device-width,user-scalable=no,initial-scale=${1/dpr},maximum-scale=${1/dpr},minimum-scale=${1/dpr}`);

    # 什么是浮动及清除浮动的方式

    • 浮动：当元素设置float属性，该元素将会脱离标准文档流。浮动在父元素左右边界内，但是由于浮动元素脱离文档流，父元素计算高度时不会将附送元素的高度计算进去，这就导致了父元素高度塌陷。但这往往不是我们想要的，我们期望元素浮动的同时，父元素计算高度还可以将浮动元素的高度计算进去。达到这个效果的过程我们就叫做清除浮动。

    • 清除浮动

      • 使用clear属性：clear属性规定那一边不允许出现浮动的元素。

        /*使用伪元素方式，需要设置 display:block
因为伪元素为行内元素，clear仅对行内元素生效*/
.container::after {
  content: '',
  display: block,
  clear: both
}

      • 使得父元素形成BFC。BFC:下个问题详解

        .container {
  overflow: hidden;
}

    # 什么是BFC，形成BFC的条件，以及BFC的特性

    • 什么是BFC

      BFC（Block Formatting Context），格式化上下文。是web页面中盒模型布局的css渲染模式。指一个独立的渲染区域或者一个隔离的独立容器。

    • 形成BFC的条件

      • 浮动元素：float除none以外的值
      • 定位元素：position(absolute/fixed)
      • display为以下其中之一的值：inline-block、table-cell、table-caption
      • overflow除了visible以外的值：hidden、auto、scroll

    • BFC的特性

      • 内部的box会在垂直方向上一个接一个的放置。
      • 垂直方向上的距离由margin决定。
      • BFC的区域不会与float的元素区域相重叠。
      • 计算BFC的高度时，浮动 元素也参与计算。
      • BFC就是页面上一个独立的容器。容器里面的子元素不会影响外面的元素。

    # transition动画与直接top，left的区别

    • transition元素会创建其单独的合成层，合成层的变化不会影响到其他层，所以可以避免回流。
    • 而top、left的变化会影响到其它元素在文档中的布局，造成回流重绘。

    # 媒体查询

    媒体查询是指针对不同的设备，特定的设备特征或者参数进行定制化的修改网站的样式

    可以通过给<link>标签加上media属性来指定该样式文件只对什么设备生效，不指定的话默认是all，即对所有设备都生效

    <link rel="stylesheet" href="styles.css" media="screen"/>
<link rel="stylesheet" href="styles.css" media="print"/>

    支持的设备类型：

    • all：适用于所有设备
    • print：适用于在打印预览模式下在屏幕上查看的分页材料和文档
    • screen：主要用于屏幕
    • speech：主要用于语音合成器

    需要注意的是：通过media指定的资源尽管不匹配它的设备类型，但是浏览器依然会加载它

    除了通过<link>让指定设备生效外，还可以通过@media让css规则在特定的条件下才能生效。响应式页面就是使用了@media才让一个页面能够同时适配PC、Pad端和手机端

    @media(min-width: 1000px) {
  /*some code*/
}

    媒体查询支持逻辑操作符：

    • and：查询条件都满足时才能生效
    • not：查询条件取反
    • only：整个查询匹配的时候才生效，常用于兼容旧浏览器，使用时必须指定媒体类型
    • 逗号或or：查询条件满足一项即可匹配

    媒体查询还支持众多的媒体特性，可以写出很复杂的查询条件：

    /*用户设备的最小高度为680px，或者为纵向模式的屏幕设备*/
@media(min-height: 680px), screen and (orientation: portrait) {}

    # 自定义属性

    自定义属性和普通属性一样具有级联性，申明在:root下的时候，在全文档范围内可用，而如果是在某个元素下申明自定义属性，则只能在它的子元素下才能使用

    自定义属性必须通过--x的格式申明，比如--theme-color: red;，使用自定义属性的时候，需要使用var函数。比如：

    /*定义自定义属性*/
:root {
  --theme-color: red;
}

/*使用变量*/
h1 {
  color: var(--theme-color);
}

    # 水平居中

    此类元素需要水平居中，则父元素必须是块级元素（block level），且父元素需要这样设置样式：

    # 固定宽高的块级盒子

    方法一：absolute+负margin

  height: 100px;

    方法二：absolute+margin auto

    .parent {
  position: relative;
}

  width: 100px;
  height: 100px;
  position: absolute;
  left: 0;
  top: 0;
  right: 0;

  position: relative;
